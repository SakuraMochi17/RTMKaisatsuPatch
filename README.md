# RTM Kaisatsu Patch

RealTrainMod (RTM) 向けの改札・運賃システムパッチです。  
Minecraft 1.7.10 / Forge 10.13.4 対応。

---

## 必要MOD
- RealTrainMod (RTM) 1.7.10.41 以上
- NGTLib 1.7.10.32 以上

---

## アイテム

| アイテム | 説明 |
|---|---|
| **乗車券** | 出発駅・到着駅が記録された一回限りの切符。入場改札でスタンプ、出場改札で回収される。 |
| **ICカード** | 残高をNBTに保存するプリペイドカード。改札で自動精算・残高表示。入場中の駅・履歴も記録。会社別カスタムデザイン対応。 |
| **定期券** | 指定区間を有効期限内なら何度でも通過できる。7日・30日・90日の3種類。途中駅も自動判定。 |
| **回数券** | 10回分の乗車を1アイテムにまとめた切符。使用のたびに残回数が減り、0になるとアイテムが消える。 |
| **一日フリーパス** | 購入当日限り何度でも通過できるパス。日付が変わると無効になる。 |
| **特急券** | 指定席券売機で購入する特急・新幹線用の切符。列車名・号車・席番・区間が記録される。指定席と自由席の2種類。 |
| **設定ツール** | ブロックを右クリックして各種設定GUIを開く。スニーク+右クリックでモデル選択。 |

---

## ブロック

### 改札機
- 設定ツールで右クリック → 駅名・モード（入場専用 / 出場専用 / 入出場兼用）・所属会社を設定
- 手に乗車券・ICカード・定期券・回数券・一日フリーパスのいずれかを持って右クリックで通過

**会社ロック**
- 改札機に会社IDを設定すると、その会社のメンバー（OP含む）しか設定操作できなくなる
- ICカードの会社が一致しない場合は通過拒否（相互利用設定で緩和可能）

**ICカードの相互利用**
- 改札機の所属会社が「相互利用許可リスト」に相手会社を登録していれば、別会社発行のICカードでも通過可能
- 設定は一方向（A→B を許可しても B→A は別途設定が必要）

**ICカードの動作**
1. 未入場状態で入場改札 → 入場記録を書き込み
2. 入場済み状態で出場改札 → 運賃を精算して残高から差引（残高不足は拒否）

**乗車券の動作**
1. 未使用で入場改札（発駅一致）→ 使用済みマーク
2. 使用済みで出場改札（着駅一致）→ 切符を消費

**定期券の動作**
- 発駅・着駅だけでなく、路線上の途中駅も有効区間として通過可能
- 有効期限切れは拒否

---

### 券売機
- 設定ツールで右クリック → 発駅・所属会社を設定
- 通常右クリックで購入GUI

**4つのタブ**
| タブ | 操作 |
|---|---|
| 切符購入 | 行き先を選んで「購入」。入場券（140円）も購入可。8件以上はスクロール対応。 |
| ICチャージ | 金額を選んで「チャージ」。ICカードをスロットに挿す。 |
| 定期券 | 行き先・期間（7/30/90日）を選んで「定期購入」。6件以上はスクロール対応。 |
| 特殊切符 | 回数券・一日フリーパスを購入。 |

**スロット（右パネル）**
- お金スロット × 4（2×2）: RTMのお金（硬貨・紙幣）を入れる
- ICカードスロット × 1: チャージ時に挿す
- お釣りは自動計算してインベントリに返却

**定期券の割引率**
| 期間 | 割引 |
|---|---|
| 7日 | 10% OFF |
| 30日 | 25% OFF |
| 90日 | 40% OFF |

**ICカードの会社別発行**
- 券売機に会社IDを設定しておくと、購入時にその会社のICカードが発行される
- 会社ごとにカード名・カラー・テクスチャを変更可能
- `assets/kaizpatch/textures/items/ic_card_<会社ID>.png` にテクスチャを置くと自動適用（なければカラー塗り）

---

### 乗越精算機
- ICカードまたは乗車券を持って右クリックすると差額精算GUI
- ICカード: 入場時の駅と現在駅の差額を自動精算
- 乗車券: 購入区間を超えた乗り越し分を精算

---

### 駅管理ブロック
- 設定ツールで右クリック → 駅名を入力してネットワークに登録
- ブロックを破壊すると駅データも削除
- 「ネットワークから削除」ボタンでブロックを残したまま登録解除可
- 累計売上（その駅の券売機での販売総額）・品目別内訳を表示
- 「売上をリセット」ボタンで累計をゼロに戻す

### 路線管理ブロック
- 設定ツールで右クリック → 路線名・初乗り運賃・距離単価・乗換料金・駅順を設定
- **スニーク + 設定ツール** で右クリック → 会社管理GUIを開く（全社横断）
- 複数路線の乗り換え運賃もDijkstra法で自動計算
- 停車駅の順番を↑↓ボタンで並び替え可能

---

### 列車管理ブロック
- 設定ツールで右クリック → 1ブロックにつき1列車を定義する

| ページ | 設定内容 |
|---|---|
| 基本情報 | 列車ID・列車名・種別（新幹線/特急/急行）・路線・指定席料金・自由席料金 |
| 停車駅 | 路線の全駅から停車駅をトグル選択（路線の駅順を維持） |
| 号車設定 | 号車ごとに座席数・クラス（普通/グリーン）を設定 |

- 「削除」ボタンで列車定義とその予約データを一括削除

---

### 指定席券売機
- 設定ツールで右クリック → 発駅設定
- 通常右クリックで購入GUI（多段階選択式）

**購入フロー**
1. 種別選択（指定席 / 自由席）
2. 行き先選択
3. 列車選択（満席は灰色）
4. 号車選択（指定席のみ・残席数表示）
5. 確認・購入（「乗車券もセットで購入」トグルあり）

**予約キャンセル・払い戻し**
- 指定席券売機の「予約をキャンセル」から購入済みの特急券をキャンセル可能
- 払い戻し額は運賃の70%（キャンセル料30%）

---

## 会社管理システム

路線管理ブロックを **スニーク + 設定ツール** で右クリックすると会社管理GUIが開きます。

| ページ | 操作 |
|---|---|
| 会社一覧 | 登録済み会社の情報表示、編集・削除・新規作成 |
| 会社編集 | 会社ID・会社名・カラー(16進)・ICカード名・デフォルト運賃を編集 |
| メンバー管理 | プレイヤー名でメンバーを追加・除名 |
| 相互利用設定 | 他社のICカード通過を許可・解除 |
| 路線紐付け | 路線をこの会社に紐付け・解除 |

**会社別運賃フォールバック**
- 路線に初乗り・加算レートが未設定の場合、所属会社のデフォルト運賃を使用
- 会社もデフォルト未設定の場合はシステムデフォルト（初乗り150円・0.1円/m）

コマンド（`/kaisatsu company` など）でも同じ操作が可能です。

---

## 運賃計算

- **距離ベース**: 駅間のブロック距離 × 距離単価 + 初乗り運賃
- **乗換料金**: 路線を跨ぐ場合は乗換先路線の `transferFee` を加算
- **IC割引**: IC精算は1円単位、切符は10円単位（切り上げ）
- 経路はDijkstra法で最安値ルートを自動選択

---

## ICカード履歴

ICカードを右クリック（またはツールチップ確認）すると、直近の乗降・チャージ履歴を確認できます。  
保存件数は最大20件。

## 空席情報Webページ

MCサーバー起動と同時に **ポート 19190** でWebサーバーが自動起動します。

| URL | 内容 |
|---|---|
| `http://サーバーIP:19190/` | 空席情報HTMLページ（10秒ごと自動更新） |
| `http://サーバーIP:19190/api/seats` | 空席情報をJSON形式で取得 |

## コマンド

すべてのコマンドは `/kaisatsu` サブコマンド体系に統合されています。

```
/kaisatsu                  ヘルプを表示（全員）
/kaisatsu web              空席情報ページURLを表示（全員）
/kaisatsu log [駅名]        改札通過ログを表示（OP）
/kaisatsu company create <ID> <会社名> <#カラー> <ICカード名>
/kaisatsu company list
/kaisatsu company delete <ID>
/kaisatsu company info <ID>
/kaisatsu company member add|remove <ID> <プレイヤー名>
/kaisatsu company mutual add|remove <自社ID> <相手ID>
/kaisatsu company fare <ID> <初乗り> <加算レート>
/kaisatsu line assign <路線ID> <会社ID>
/kaisatsu line list
```

`company` / `line` サブコマンドはOP権限が必要です。  
GUIからも同じ操作が可能です（路線管理ブロックをスニーク+設定ツールで右クリック）。
